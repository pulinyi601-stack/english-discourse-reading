## 一、产品整体架构（全局）

### 1.1 页面整体分区

页面分为三大固定区域，UI严格参照现有线上原型：

1. **顶部导航区**：主题色切换（唐柳绿/海雾蓝/暖纸棕/樱粉柔光/午夜黑）、侧边菜单入口、教师智能体标识

2. **左侧功能菜单栏**：辅助教学功能区、辅助命题功能区、历史记录、自由对话四大板块

3. **主内容区**：语篇研读报告输入框、生成按钮、报告展示面板、历史记录切换面板

### 1.2 全局通用规则（全页面生效）

- 未登录游客：文本临时缓存24h，关闭页面不保存历史记录，每日免费生成报告3次

- 登录用户：永久保存历史记录，支持报告回看、删除、批量管理

- 内容风控：所有输入英文文本前置安全审核，拦截违规、色情、暴力内容，直接弹窗拒绝生成

- UI要求：适配PC端为主（教师办公场景），保留移动端自适应，5套主题色一键切换无布局错乱

## 二、核心功能模块需求（分前端/后端/算法，明确开发职责）

### 模块1：顶部全局主题切换 + 侧边菜单（前端独立完成）

#### 功能清单

1. 5套预设主题色一键切换，切换实时生效，无需刷新页面

2. 左侧侧边栏可收起/展开，菜单二级目录默认折叠，hover展开

3. 菜单两大板块固定：辅助教学功能区、辅助命题功能区，所有菜单按钮仅做页面跳转，本期V1.0只上线【语篇深度研读报告】核心功能，其余菜单仅占位

#### 交互规则

菜单点击记录点击状态，高亮当前所在页面；历史记录面板右侧悬浮，可一键清空全部记录

### 模块2：语篇深度研读报告（核心核心功能，全端联动）

#### 2.1 输入交互（前端）

- 大文本输入框：支持手动粘贴英文语篇，实时统计总词数、总句数

- 输入校验：非英文文本直接前端弹窗提示，请输入标准英文语篇

- 按钮：【生成研读报告】，点击后加载中状态，禁止重复点击防重复提交

#### 2.2 后端预处理逻辑（后端）

1. 文本清洗：自动去除多余换行、空格、特殊乱码符号

2. 文本分片：超过2000词自动分片处理，避免大模型上下文溢出

3. 任务管理：生成唯一task_id，异步任务排队，前端轮询请求生成进度

4. 额度校验：调用接口校验用户剩余免费/付费额度，额度不足弹窗引导充值

#### 2.3 算法NLP+大模型生成逻辑（算法端）

固定输出框架：强制按照What-Why-How三层结构生成报告，禁止AI自由发散，报告固定6大模块，缺一不可

1. **What 文本基础信息**：文本主题、核心内容概括、段落大意拆分

2. **Why 写作意图分析**：作者写作目的、情感态度、核心观点、文本育人价值（贴合高中英语课标）

3. **How 文本写法分析**：文体结构判定、行文逻辑、句式特点、修辞分析、重难点词汇&长难句标注

4. 教学延伸：课文教学重难点、阅读命题切入点、课堂教学设计建议

#### 2.4 报告展示与操作（前端）

- 报告以Markdown格式渲染，层级清晰，标题、列表格式统一

- 支持功能：一键复制报告文本、**导出Word文档、PDF文档、Markdown源码**、保存至个人历史记录

- 原文联动：报告中标注的长难句/重点词汇，hover可悬浮展示原文对应位置

#### 2.5 数据支撑要求

- 内置CEFR词汇分级库：自动匹配词汇A1-C2难度等级

- 高中英语课标RAG知识库：所有分析贴合国内高中英语教学要求，杜绝西式主观解读偏差

- 固定Prompt模板：分文体（记叙文/议论文/说明文/新闻）专属提示词，保证输出格式统一

### 模块3：历史记录管理（前端+后端）

- 记录内容：文本标题、生成时间、报告缩略内容、操作按钮（查看/删除）

1. 大模型对接：统一调用**DeepSeek V4 Pro**官方API接口

2. 模型专属适配：依托DeepSeek V4 Pro超长上下文能力，原生支持3000词内英文文本无需分片；超3000词再启动后端自动分片+全文汇总逻辑

3. 防幻觉强制要求：所有分析内容必须绑定原文语句，禁止AI无依据编造内容；DeepSeek V4 Pro强指令遵循能力适配固定报告模板，输出稳定性更高

4. 缓存策略：高中高频课文分析结果永久缓存，重复调用0token消耗，降低接口成本

- 登录用户：记录永久保存，支持单条删除、一键清空全部记录

- 回溯功能：点击历史记录，一键复原原文+完整报告，无需重新生成

### 模块4：其余菜单占位功能（V1.0暂不开发，仅UI占位）

以下功能本期只保留菜单入口，点击跳转空白占位页，二期迭代开发：

1. 辅助教学：言说策略分析、课文语言点讲解、词块教学、作文批改、试卷讲评、读写教学设计、图片生成

2. 辅助命题：阅读设问、文本改编、完形命题、试题Bug检测、超标词排查替换

3. 自由对话：AI自由问答入口

## 三、全链路技术架构要求（开发统一标准）

### 3.1 前端技术栈

Vue3 + Vite + Element Plus + Markdown-it + ECharts；要求：页面加载速度≤2s，主题切换无卡顿，异步加载loading状态完善

### 3.2 后端技术栈

FastAPI + MySQL + Redis + Celery

- MySQL：存储用户信息、额度、订单、历史记录

- Redis：缓存热门文本分析结果、用户临时文本、异步任务队列

- Celery：处理大文本异步分析、报告PDF导出耗时任务

### 3.3 算法&大模型对接要求

1. 大模型对接：统一调用教育类微调LLM接口

2. 防幻觉强制要求：所有分析内容必须绑定原文语句，禁止AI无依据编造内容

3. 缓存策略：高中高频课文分析结果永久缓存，重复调用0token消耗，降低接口成本

## 四、常见问题及固定解决方案（开发提前规避，无需额外讨论）

|线上潜在问题|标准化解决方案|
|---|---|
|大文本超出模型上下文窗口，报告截断|后端自动分片分析，最后大模型统一汇总全文报告|
|AI报告格式混乱、缺失分析模块|后端增加格式校验脚本，模块缺失自动重跑一次请求|
|恶意用户刷免费额度|设备指纹+IP双重限流，每日免费次数硬限制|
|Word/PDF/Markdown多格式导出排版错乱|禁止前端本地导出，后端统一生成标准化Word、PDF、Markdown三种格式文件，固定排版模板，保证多端打开格式无错乱|
|文本含有学生隐私信息|后端自动脱敏，屏蔽手机号、姓名、学校等敏感字段|

## 七、后端统一接口数据规范（Claude Code 可直接一键开发，无额外沟通成本）

### 7.1 接口通用全局规范（所有接口统一遵循）

- **请求前缀**：/api/v1/english/

- **请求头统一参数**：Authorization（登录用户传Token，游客为空）、Content-Type: application/json

- **通用返回体格式（所有接口强制统一）**

```json
{
  "code": 200, // 状态码：200成功 / 400参数错误 / 401未登录 / 403额度不足 / 500服务异常
  "msg": "请求成功", // 提示文案，前端直接弹窗展示
  "data": {} // 业务返回数据，无数据返回空对象
}
```

- **异步任务统一规则**：长文本分析、文件导出均走异步任务，前端轮询task_id查询进度

- **字符编码**：全局UTF-8，兼容英文全文、特殊标点、长难句符号

### 7.2 核心接口明细（5个必做接口，覆盖全部业务）

#### 接口1：文本前置校验+清洗接口（前端输入文本后第一步调用）

- 接口地址：/text/check

- 请求方式：POST

- 接口作用：校验是否为英文、清洗文本、统计词数句数、前置风控内容审核

##### 请求入参

```json
{
  "raw_text": "string", // 必填：用户输入原始英文文本
  "user_type": "visitor/login" // 必填：visitor游客 / login登录用户
}
```

##### 返回出参data字段

```json
{
  "is_valid_english": true, // 是否为有效英文文本
  "clean_text": "string", // 清洗后标准文本（去除空格/乱码）
  "word_count": 1256, // 总词数
  "sentence_count": 48, // 总句数
  "risk_status": "pass/reject", // 内容风控结果：pass通过 / reject违规拦截
  "risk_msg": "" // 违规提示文案
}
```

#### 接口2：发起语篇研读报告生成任务（核心业务接口）

- 接口地址：/report/generate

- 请求方式：POST

- 接口作用：校验用户额度、创建异步分析任务、返回任务ID

##### 请求入参

```json
{
  "clean_text": "string", // 必填：清洗后的标准英文文本
  "text_word_num": 1256, // 必填：文本总词数
  "user_token": "string", // 选填：登录用户token，游客填空
  "user_type": "visitor/login"
}
```

##### 返回出参data字段

```json
{
  "task_id": "task_20260622_0001", // 唯一异步任务ID，前端用于轮询
  "remain_quota": 3, // 用户剩余免费生成额度
  "task_status": "pending" // 任务状态：pending进行中 / success成功 / fail失败
}
```

#### 接口3：查询报告生成异步任务进度（前端轮询接口）

- 接口地址：/report/progress

- 请求方式：GET

- 接口作用：前端每2s轮询一次，获取报告生成进度与最终结果

##### 请求入参（URL参数）

```text
?task_id=task_20260622_0001
```

##### 返回出参data字段

```json
{
  "task_status": "pending/success/fail",
  "progress_percent": 65, // 生成进度百分比 0-100
  "report_content": "# 语篇研读报告\n### What...", // 成功后返回完整markdown格式报告原文
  "fail_reason": "" // 失败原因，成功返回空
}
```

#### 接口4：报告多格式文件导出接口（支持Word/PDF/Markdown三类格式）

- 接口地址：/report/export

- 请求方式：POST

- 接口作用：后端统一生成文件，前端直接下载，禁止前端本地导出

##### 请求入参

```json
{
  "task_id": "task_20260622_0001",
  "export_type": "word/pdf/markdown", // 固定三选一，严格匹配
  "report_content": "string" // 完整报告markdown原文
}
```

##### 返回出参data字段

```json
{
  "file_download_url": "https://xxx.oss.com/report/xxx.docx", // 文件直链
  "file_name": "高中英语语篇研读报告_20260622" // 默认下载文件名
}
```

#### 接口5：用户历史记录增删查接口

- 接口地址：/history/operate

- 请求方式：POST

- 接口作用：新增记录、查询列表、单条删除、清空全部记录

##### 请求入参

```json
{
  "operate_type": "add/list/delete/clear", // 操作类型
  "task_id": "task_20260622_0001", // 删除/新增时必填
  "user_token": "string" // 登录用户必填，游客无历史记录
}
```

##### 返回出参data字段

```json
{
  "history_list": [
    {
      "task_id": "task_20260622_0001",
      "text_preview": "文本前50字预览",
      "create_time": "2026-06-22 15:30:22",
      "report_preview": "报告摘要预览"
    }
  ],
  "operate_result": true // 增删清空操作是否成功
}
```
